Why an LFM — privacy is the unlock

The whole brain runs on the device.
Only a family alert crosses the line.

Privacy / sovereignty. A camera + mic in an elder's home is only acceptable if data never leaves it. A cloud LLM cannot meet that bar — an on-device LFM can.

Always-on, low cost. Continuous pose check is cheap MediaPipe; the LFM2-VL vision model is invoked on ~1 frame only when a fall is suspected. Low-power, no per-frame cloud bill.

Offline resilience. Works in airplane mode — conversation, fall detection and the 119 script all keep running with no network.

Domain fit. A 1.2B Japanese LFM tuned for gentle, slow elder-care dialogue beats a generic large model on this specific task.

Results — fall → graduated escalation

From a quiet fall to a 119 hand-off, automatically.

S1 · CHECK-IN

Speak to the resident

VLM confirms a person lying down → Tomoshibi gently asks "Are you OK?" (🙆 resolves).

S2 · FAMILY (after 15s)

Notify family

No response → alert the registered family contact. The only message that leaves the device.

S3 · 119 (after 5s)

Read the dispatch script

LFM2 generates the 119 script live — name, conditions, medications, allergies — spoken aloud. 🆘 jumps straight here.
